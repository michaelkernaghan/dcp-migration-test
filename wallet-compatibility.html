MIG-60: Mobile UI Layout Issue
Issue Description

On Android or Apple mobile devices, when users navigate to decentralized.pictures, the "Submit" button overlaps and partially hides the password input field, making it difficult for users to see what they're typing.

Technical Details
  • Error Type: UI layout and positioning issue
  • Affected Platforms: Android mobile, Apple mobile (iOS)
  • User Impact: Submit button obscures password input field
  • Frequency: High - affects all mobile users on these platforms
Expected Behavior

The button should be fully visible and positioned below the input field, with appropriate padding.

Current Status
Medium In Progress Issue being investigated by development team
Issue Metadata

Priority: Medium
Status: In Progress

Workaround

Until this issue is resolved, mobile users can try:

  • Rotating the device to landscape mode to see more of the interface
  • Using the desktop version of the site if available
  • Zooming out to see the full form layout
MIG-61: Mobile Wallet Connection Error Message
Issue Description

On mobile devices, when users attempt to connect a wallet through the Advanced Login option, they see a confusing red error message: "You have cancelled the claiming cycle" even when no actual error occurred or when the user simply didn't complete the wallet connection process.

Steps to Reproduce
  1. Open the app/site on a mobile device
  2. Tap Advanced Login
  3. Select a wallet from the list
  4. Observe the red error message
Expected Result

A more accurate or user-friendly message (e.g., "Wallet connection was cancelled" or "Login process interrupted") should be shown, or the issue should be prevented if no real error occurred.

Actual Result

The user sees: "You have cancelled the claiming cycle."

Current Status
Medium Under Investigation Issue being analyzed for root cause
Issue Metadata
Jira ID: MIG-61
Priority: Medium
Status: Under Investigation
Assignee: Development Team
Created: January 2025
Temporary Solution

Users experiencing this error message can try:

  • Ignoring the error message and retrying the wallet connection
  • Using a different wallet option if available
  • Refreshing the page and attempting the connection again
  • Using the desktop version of the site if the issue persists

Ledger Testing Flow for TALNT Migration
Pre-Migration Setup
  1. Connect Ledger Nano S via USB
  2. Unlock device with PIN
  3. Open Ethereum app on device
  4. Enable "Contract Data" in app settings
  5. Connect to MetaMask or Ledger Live
  6. Add Base Appchain network manually
Migration Process
  1. Authenticate with Google/X (if applicable)
  2. Connect hardware wallet to DCP platform
  3. Verify Tezos address matches Ledger
  4. Review migration summary
  5. Confirm transaction on Ledger device
  6. Verify TALNT tokens in Base Appchain wallet
Critical Test Points:
  • Device connection stability during long processes
  • Transaction approval with custom contract data
  • Network switching between Tezos and Base Appchain
  • Error handling when device disconnects
  • Recovery flow if transaction fails mid-process
