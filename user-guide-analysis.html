User Guide Cross-Check Analysis

Role-by-role analysis comparing Character Profiles from the migration test site with the TALNT Token Migration User Guide PDF.

This analysis identifies potential user confusions and evaluates how well the PDF documentation addresses each test persona's concerns.

Kanley Stubrick – Visionary Founder

Google sign-in Moderator High Reputation (40.4)

Likely User Confusions
  • Does a social-login moderator need to install a wallet?
  • Will moderator rights, reputation 40.4 and NFT #4454 survive?
  • Will FILM→TALNT keep the same value?
How PDF Addresses These Concerns

Social-login track: "The app handles all the wallet management automatically – no MetaMask installation or wallet connections required."

Reputation & NFT preservation: Explicitly promised in Account Migration ("preserve your reputation scores and history") and NFT Migration bullets.

Gap: Role privileges (e.g. moderator) are not mentioned, so Kanley's concern about losing special permissions is not yet covered.

Herner Werzog – Security-minded Submitter

Google sign-in 2FA Enabled

Likely User Confusions
  • Will 2FA interfere with the one-click social flow?
  • ID-verification flagged "Failed" – does that block migration?
How PDF Addresses These Concerns

The Social-login path reassures that the same Google account is enough and shows a four-step guided claim with "no wallet setup required."

Gap: 2FA itself is never discussed, and the FAQ doesn't cover failed ID-verification, so these edge-case worries remain open.

TwitterProd – Social Media User

X (Twitter) sign-in No email on file

Likely User Confusions
  • If notifications go by email, will a user with no email learn the claim window is open?
  • Does an X-only account need anything beyond the X OAuth flow?
How PDF Addresses These Concerns

The PDF assumes notifications can reach the user ("you'll receive notification…") but doesn't state the channel; it relies on the banner that appears after log-in.

Since TwitterProd can still log in with X, the in-app banner covers the process.

Gap: Email-specific reminders are not addressed for users without email addresses.

Keith Fellini – Baker / Validator

Sentinel wallet (Tezos) EVM wallet High Balance (3,275 TALNT)

Likely User Confusions
  • Must I connect two wallets?
  • Are my 3,275 TALNT safe?
  • Mixed 2FA — will hardware signing pop-ups work?
How PDF Addresses These Concerns

Dual-wallet flow: External-wallet user track spells out the dual-wallet flow (connect Tezos + EVM), shows the progressive claim dialog.

Token safety: Confirms 1:1 FILM→TALNT conversion. Gas-fee and "What if something goes wrong?" sections give comfort about large balances and retries.

Gap: 2FA specifics are not covered but normal wallet signing is assumed.

Sandra Blockchain – Hardware Wallet User

Ledger Nano S No DCP account 3 NFTs Locked balance (100)

Likely User Confusions
  • Where does a wallet-only holder (no platform login) start?
  • Will multiple NFTs and a 100-token lock migrate?
  • Does the Ledger hardware wallet work with Base?
How PDF Addresses These Concerns

Wallet-only users: "For Wallet-Only Token Holders" section tells such users to use a dedicated migration portal and mirrors the same four-step claiming flow.

NFT preservation: The NFT bullet promises metadata and ownership preservation.

Gap: Locked-token mechanics and Ledger compatibility are not spelled out, so Sandra may still need a FAQ entry.

New User – Post-migration Sign-up

No existing account First-time user

Likely User Confusions
  • How do I register on the new chain?
  • Do I still get welcome tokens?
How PDF Addresses These Concerns

The guide focuses on migration cohorts. It hints at "new opportunities for filmmakers" but offers no dedicated onboarding steps for first-time users.

Gap: This persona is completely uncovered in the current PDF.

Summary of Documentation Gaps

Role-based Privileges

Moderator/baker status and special permissions migration is not addressed in the PDF.

Affects: Kanley Stubrick, Keith Fellini

2FA Edge Cases

Two-factor authentication interactions with migration flow are not documented.

Affects: Herner Werzog, Keith Fellini

Hardware Wallet Compatibility

Ledger compatibility with Base chain and locked token mechanics need clarification.

Affects: Sandra Blockchain

Email-less Users

Users without email addresses may miss important migration notifications.

Affects: TwitterProd

New User Onboarding

Post-migration onboarding process for first-time users is completely missing.

Affects: New User persona

Recommended Actions
  • Add an explicit "New to DCP after June 17?" onboarding section
  • Include 2FA interaction guidelines for both social and wallet authentication
  • Clarify role/privilege preservation during migration
  • Add hardware wallet compatibility matrix for Base chain
  • Address notification delivery for users without email addresses
